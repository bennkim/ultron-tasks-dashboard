## ICE 스코어링

모든 태스크에 **ICE 점수**를 부여하여 우선순위 판단:

| 항목 | 설명 | 범위 |
|------|------|------|
| **I** (Impact) | 비즈니스 영향도 | 1-10 |
| **C** (Confidence) | 성공 확신도 | 1-10 |
| **E** (Ease) | 실행 용이성 | 1-10 |
| **Score** | I × C × E | 1-1000 |

**우선순위 기준:**
- 🔴 700+ → 즉시 실행
- 🟠 400-699 → 이번 주
- 🟡 200-399 → 이번 달
- ⚪ 1-199 → 백로그

태스크 테이블 예시:
```
| Task | 상태 | 담당 | 설명 | ICE |
| T-001 | ✅ | CTO | Pixel 삽입 | 9×8×7=504 |
```

## 규칙

### 태스크 생성
- 새 안건 → Epic 생성 (E-XXX)
- Epic 아래 Story 분류 (S-XXX)
- Story 아래 실행 단위 Task (T-XXX)
- 번호는 순차 증가, 절대 재사용 안 함

- 담당자가 착수 시 → IN_PROGRESS
- 블로커 발생 시 → BLOCKED + 블로커 명시
- 완료 시 → DONE + 날짜
- 의장 확인 필요 시 → REVIEW
